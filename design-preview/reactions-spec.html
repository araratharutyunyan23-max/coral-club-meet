In-call reactions

Anyone taps an emoji; it floats up over the stage with the sender’s name, then fades — broadcast to everyone on the data channel. This redesign makes that moment unmistakably Coral Club: the reaction emits a Signal-Field ripple — coral core, teal water — and rises trailing the sender’s own per-user hue, its name set in a small glass pill. Calm-premium, GPU-cheap, one-shot ~3s. Everything below drops into theme.css and a presentational <ReactionsOverlay>.

REC

Ship Signal-burst as the default.

It’s the only option that speaks the brand’s core language: a reaction literally becomes a signal on the pond — the same coral-core / teal-ring geometry as the ambient Signal Field, now emitted by a person. Sparkle-trail and Glass-badge are the alternates — one warmer/playful, one maximally legible — behind a single REACTION_STYLE constant so the whole product stays consistent. All three share the same physics, spawn zone, name pill and reduced-motion fallback.

A

Tokens — add to theme.css

Four properties drive the whole reaction. The ripple bloom is just --coral at ~85% — the same Signal-Field ring, now emitted by a person — with the outer ring reusing --ripple-ring (teal water) verbatim. Only the bloom softens on the pale Tide surface. The sender’s hue is not a token — it’s computed per reaction (§E).

B

Anatomy, z-index & spawn zone

  • Two anchors, one origin. Each reaction is a static .rx-bloomslot (the ripple, stays put) + a .rx-slot that rises carrying emoji, trail and name pill. Both are absolutely placed at the same (left%, bottom px) origin — same “single origin” rule as the ambient system.
  • Never blocks the call. The whole .rx-layer is pointer-events:none and sits below the top bar and control bar (z 4 < 6), so it reads over the video but every control stays clickable.
  • Out of the centre & speaker. Singles spawn in the left gutter (7–26%); clusters use both gutters (7–30% · 70–92%) and never the centre column (34–66%) where the active speaker sits. Vertical start is 120–184px, clear of the control bar.

C

Keyframes — bloom · rise · glow · sparkle · name

The one rule that keeps it safe everywhere: only transform and opacity ever animate (no width/top/filter tweening) — GPU-composited, zero layout thrash even when a dozen fire at once. Every animation runs once (both/forwards); the node is removed on completion so nothing accumulates.

D

Cluster — when several react at once

A group reaction should feel alive together, not a random scatter. When N reactions arrive inside a short window, spawn them as a staggered fan (~70ms apart) across both gutters, each slightly larger — they read as one swell rising, not confetti. The debounce is per-emoji-burst on the client, not new network state.

E

Name pill & the per-user-hue rule

The name is set in a glass pill — --glass + backdrop-filter so it stays legible over any video — with the sender’s per-user hue as a small accent dot (Glass-badge uses the hue as a left bar instead). The hue comes from the exact same hueFor() the Avatar atom uses, so a person’s reaction, tile and avatar all share one colour. The local sender always reads “You.”

F

Reduced motion — a calm fade, no particles

Under prefers-reduced-motion: reduce (or the product’s data-motion="off") the layer gets .rm: no ripple bloom, no glow, no sparkles, no travel. The emoji and its name pill simply fade up 8px and settle in place, hold, and fade — same information, zero motion. Clusters drop to 3 well-spaced pieces (§D).

G

Themes & performance

  • Both themes for free. Pill, chip, bloom and text all reference --glass, --coral, --ripple-ring, --text — which already flip. The single manual tweak is the softened --rx-bloom on Tide.
  • GPU-cheap by construction. Only transform/opacity animate; every animated node carries will-change:transform,opacity; nothing touches layout. A full cluster is ~18 short-lived compositor layers.
  • Self-cleaning. Each node is removed on its longest animationend (or a dur + buffer timer) — the DOM never grows, so a long call with thousands of reactions stays flat.
  • Legible over anything. Emoji carry a soft drop-shadow; the name always sits on glass — so a reaction reads over a bright screen-share or a dark tile equally.

H

Drop-in — ReactionsOverlay.tsx

One presentational component. It subscribes to reaction events on the data channel, keeps a short-lived list, renders a node per reaction, and drops each from state when it finishes. The sender picks only an emoji — style, hue, spawn and physics are all derived, so every client renders the same thing.
